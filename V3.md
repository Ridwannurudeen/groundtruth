# GroundTruth V3 — Belief-Maintenance Layer (the A→Z build)

**Thesis upgrade.** V1/V2 built "an app that deletes retracted papers." V3 builds what that was always pointing at: a **belief-maintenance layer for agent memory** on Cognee. Memory is not a pile of facts — it is a set of **beliefs** with evidence, confidence, and a lifecycle (`active → contested → superseded/retracted → purged`), kept consistent as new evidence arrives, session after session. Retraction is just the trivially-verifiable special case.

**Roles.** Codex builds this spec. Claude audits when done — every phase must leave **auditable evidence** (real transcripts, real numbers, reproducible commands in `docs/RESULTS-V3-P<n>.md`). Expect the audit to recompute your metrics from raw data and read your code for shortcuts; build accordingly. Honest failure documented > silent shortcut. The V2 audit caught a 13-of-25 gap that inflated a headline — that class of finding must be zero this time.

**Current state (all verified, commit `859985b` on `plan/build-package`, 28 tests green):**
- Working: per-claim `remember` ingestion (one claim = one `data_id`), retraction watcher (RW DOI match → `contradicts`/`supersedes` edges stamped via `upsert_edges` → surgical `forget(memory_only=True)`), graph-in-loop `answer()` (metric from real `GRAPH_COMPLETION` `objects_result`, token-match demoted to cross-check), LLM semantic judge (`LLMGateway.acreate_structured_output`), 28-pair labeled eval (partial: 21 judged, P 1.00 / R 0.67, 7 pending on quota), FastAPI demo, honest disclosures.

## P1 — Belief-state core

**Files:** extend `groundtruth/registry.py`, `groundtruth/ontology.py`, `groundtruth/answer.py`; new `groundtruth/beliefs.py`.

1. **Belief lifecycle** on every claim in the registry: `belief_state ∈ {active, contested, superseded, retracted, purged}` + `state_history: [{state, at, evidence_class, evidence_ref, basis}]`. Migrate existing claims (`retracted_forgotten` → `retracted`; controls → `active`).
2. **Evidence ladder** — every state change records its evidence class: `authority_feed` (deterministic, e.g. Retraction Watch) > `semantic_inference` (LLM judgment + confidence) > `user_assertion` (adjudication/feedback). This is the epistemology-as-architecture feature: no belief changes without a classed, referenced piece of evidence.
3. **Graph mirror:** state expressed via existing edges (`supersedes`, `superseded_by`, `contradicts`) plus **NodeSet tagging** — `belongs_to_set` is native on DataPoint (verified: `cognee/infrastructure/engine/models/DataPoint.py:62`); tag claims into node sets by state (e.g. `beliefs_contested`) where practical at ingest/re-ingest time.
4. **Recall annotation:** `answer()` annotates every reference with its belief state + the latest state-change basis. `cites_retracted` generalizes to `cites_by_state` counts.
   **Gate:** answer payload shows per-reference belief state; registry state_history populated for all 40 claims; suite green.

## P2 — Contested-belief workflow (makes the feedback loop load-bearing)

**Files:** new `groundtruth/contest.py`; extend `watcher.py`, `web/app.py`, UI.

1. **Confidence bands** for Tier-B semantic judgments: `≥ 0.9` auto-act (supersede per direction); `0.7–0.9` → mark **contested** (both claims stay in memory, both annotated in answers: "⚠ contested by <claim>, basis: …"); `< 0.7` log-only.
2. **Adjudication queue:** `GET /contested` lists contested pairs; `POST /adjudicate {pair, verdict}` resolves — verdict source recorded as `user_assertion`, or auto-resolved later if an authority feed (P5) covers it. Resolution transitions belief states and (only then) triggers supersede/forget.
3. **Feedback wiring (real, verified APIs):** adjudications and answer thumbs go through `cognee.session.add_feedback(session_id, qa_id, feedback_text, feedback_score)` (`cognee/api/v1/session/session.py:77`), then `improve(dataset, session_ids=[...], build_truth_subspace=True)` (`improve.py:36`), then recall with `feedback_influence>0` (`recall.py:379`; default 0). Demonstrate one measurable effect: a source's rank shifts after adjudication+improve; if the effect is too subtle to show, say so in RESULTS — do not fake it.
   **Gate:** full contested lifecycle transcript: detect → contested (answers show the warning) → adjudicate → state transition → answers change. Suite green.

## P3 — The Morning-After Briefing agent (the hackathon-narrative killer)

**Files:** new `groundtruth/briefing.py` (+ CLI), extend `web/`.

Canonical Cognee self-improvement flow (their promoted quickstart), in our domain:
1. **Evening session:** evidence arrives session-only — `remember(notice_text, dataset_name=..., session_id="analyst-<date>", self_improvement=False)`.
2. **Recall before:** permanent memory still confidently cites the soon-to-be-dead claim (capture verbatim — this is the "amnesia" beat).
3. **Bridge:** `improve(dataset=..., session_ids=["analyst-<date>"])` persists session content into the permanent graph (verified: improve docstring stages).
4. **Reconcile:** watcher + Tier-B run over the newly-bridged evidence → contradiction/contest/supersede/forget per P1/P2 rules.
5. **The briefing:** generate `MORNING_BRIEFING.md` (and `GET /briefing`): *learned last night / now contested / revised (with evidence class + basis) / purged*. Same question re-asked → corrected, annotated answer.
   **Gate:** one complete scripted run with verbatim before/after answers + the briefing artifact. This transcript is the demo's spine — make it clean enough to screen-record.

## P4 — Belief timeline & time-travel diff

**Files:** new `groundtruth/timeline.py`; extend `web/`.

1. **Deterministic time-travel** from `state_history` + supersession chain: `answer_as_of(question, date)` reconstructs which claims were believed at `date` and answers from that set (filter references by state-at-date; annotate "as-of" clearly).
2. **Diff view:** `GET /timeline?from=X&to=Y` → beliefs added / contested / revised / purged between two dates, each with evidence class and basis. UI: side-by-side "answer then vs answer now" diff.
3. Optional garnish only if cheap: `temporal_cognify=True` + `SearchType.TEMPORAL` (verified: event-time extraction from content, `temporal_retriever.py`) — but the deterministic registry path is the load-bearing implementation; do not block on TEMPORAL.
   **Gate:** "What did we believe about <claim topic> in 2023 vs today?" produces a correct, visibly different pair of answers with the chain shown.
